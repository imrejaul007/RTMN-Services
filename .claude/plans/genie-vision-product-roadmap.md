# The App Itself — UX Design

## Why UX matters

Most founders think about services and agents. Users think about **screens, buttons, widgets, notifications, and daily habits**.

The current Genie architecture is strong, but the **app experience is underdefined**.

The app should feel like:

> **ChatGPT + Apple Health + Google Photos + Notion + Calendar + Rewind + Wealthfront + Replika**
>
> unified into one personal operating system.

## The 5-Tab Rule

I would not build 23 menu items. I would build **5 primary tabs**:

```
🏠 Home
💬 Genie
🔍 Search
📚 Memory
👤 Me
```

That's it. Everything else is inside.

# What This Means for Engineering

The backend is mostly already built (23 specialist services wired through runtime/genie). What we need next:

1. **A polished consumer mobile app** (PWA first, then native). The do-app Expo stack is a great starting template.
2. **5-tab navigation** following the design above.
3. **Voice-first interactions** — push-to-talk, "Hey Genie" wake word, screen awareness.
4. **Lock screen widgets** — morning briefing, daily score.
5. **Notification discipline** — useful signals, not spam.
6. **Connected accounts** — Google, Apple Health, WhatsApp, bank aggregators.
7. **Polished onboarding** — the first 30 seconds that teaches the user "Genie knows everything about your life, just ask."

The strategic bet: **The app experience is the moat**. The backend capabilities are largely commoditized. Whoever wins builds the daily habit.
