## What this app does

TradeFlow is a B2B import/export marketplace for the Middle East region.
Buyers post RFQs (Requests for Quotation) for products they need.
Sellers respond with quotes. SUTAR AI agents handle the entire workflow:
procurement, negotiation, trade finance, logistics, customs, and settlement.

The app supports English and Arabic, handles multi-currency (USD, AED, INR),
and is registered as a federated Nexha on Global Nexha so it can discover
and trade with other Nexhas worldwide.

## Architecture

- **Backend:** Node.js (v20+) + Express + TypeScript (strict mode)
- **Frontend:** Next.js 14 (App Router) + TypeScript + Tailwind CSS
- **Mobile:** Flutter 3.x (iOS + Android)
- **Database:** MongoDB 7.x (via Mongoose)
- **Cache:** Redis 7.x

## How to Extend

### Add a new SUTAR agent

1. Create `apps/backend/src/agents/myagent.ts`:

```typescript
import { BaseAgent } from '@hojai/sutar';
import { myService } from '../services/myservice';

export class MyAgent extends BaseAgent {
  constructor() {
    super({
      role: 'my-role',
      capabilities: ['my-capability'],
      tools: ['@hojai/commerce', '@hojai/nexha'],
      memory: '@hojai/foundation'
    });
  }

  async execute(task: Task): Promise<Result> {
    // Your agent logic here
    return await myService.doSomething(task);
  }
}
```

2. Register in `apps/backend/src/agents/index.ts`:

```typescript
import { MyAgent } from './myagent';

export const agents = {
  ceo: new CEOAgent(),
  sales: new SalesAgent(),
  // ... existing agents
  myAgent: new MyAgent()
};
```

3. Wire to CEO Agent's coordination if needed

### Add a new API endpoint

1. Create `apps/backend/src/routes/myroute.ts`:

```typescript
import { Router } from 'express';
import { z } from 'zod';
import { requireAuth } from '../middleware/auth.middleware';
import { myService } from '../services/myservice';

const myRouter = Router();

const querySchema = z.object({
  // Define query params
});

myRouter.get('/',
  requireAuth,
  async (req, res) => {
    const params = querySchema.parse(req.query);
    const result = await myService.doSomething(params);
    res.json({ success: true, data: result });
  }
);

export { myRouter };
```

2. Mount in `apps/backend/src/index.ts`:

```typescript
import { myRouter } from './routes/myroute';
app.use('/api/v1/myroute', myRouter);
```

### Update capability declaration (publish to Global Nexha)

1. Edit `.hojai/capability.json`
2. Run `npm run capability:publish` (or `bash scripts/capability:publish.sh`)
3. CapabilityOS is updated; DiscoveryOS will pick up the new capabilities within 5 minutes

### Add a new page (web)

1. Create folder `apps/frontend/app/mypage/`
2. Add `page.tsx`:

```typescript
export default function MyPage() {
  return (
    <div>
      <h1>My Page</h1>
    </div>
  );
}
```

3. Connect to API via `apps/frontend/lib/api.ts`

### Add a new screen (mobile)

1. Create `apps/mobile/lib/screens/myscreen.dart`
2. Add route in `apps/mobile/lib/main.dart`
3. Connect to API via `apps/mobile/lib/services/api.dart`

### Update Nexha federation settings

1. Edit `.hojai/capability.json`
2. Run `npm run nexha:federate` (or `bash scripts/federate.sh`)
3. Federation handshake sent to Global Directory

## Conventions

- **Validation:** Use **Zod** schemas for all inputs
- **Auth:** Use `requireAuth` middleware on all `/api/v1/*` routes (except public webhooks)
- **Errors:** Throw typed errors; `errorMiddleware` handles HTTP responses
- **Logging:** Use `logger.info/error` from `@hojai/foundation`
- **Events:** All cross-agent communication via SUTAR's event bus
- **Tests:** **Vitest** for unit, **supertest** for API integration
- **TypeScript:** Strict mode, no `any`
- **Code style:** ESLint + Prettier (configs in `packages/config/`)
- **Commits:** Conventional commits (feat:, fix:, chore:, etc.)
- **Branches:** `main` (production), `develop` (integration), `feature/*`
