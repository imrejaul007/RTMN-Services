### 2.2 SkillOS powers HOJAI Foundry (developers)

Developers using `@hojai/foundation` SDK can register and use skills:

```typescript
import { SkillOS } from '@hojai/skillos';

// Register a custom skill
await SkillOS.register({
  name: 'custom-recommendation',
  category: 'AI',
  description: 'Personalized product recommendations',
  input: { userId: 'string', context: 'object' },
  output: { recommendations: 'array' },
  implementation: async (input) => {
    // Custom logic
  }
});

// Discover skills
const skills = await SkillOS.discover({ category: 'Commerce' });

// Execute a skill
const result = await SkillOS.execute('whatsapp-send', {
  to: '+1234567890',
  message: 'Your order has shipped!'
});

// Compose multiple skills into a workflow
const workflow = await SkillOS.compose({
  name: 'order-fulfillment',
  steps: [
    { skill: 'inventory-check', input: { sku: 'X' } },
    { skill: 'payment-process', input: { orderId: 'Y' } },
    { skill: 'whatsapp-send', input: { message: 'Shipped!' } }
  ]
});
```

### 2.3 SkillOS powers SUTAR (agent runtime)

Every SUTAR agent uses SkillOS to extend its capabilities:

```typescript
class ProcurementAgent extends BaseAgent {
  async execute(task) {
    // Use built-in skills
    const suppliers = await this.skills.execute('supplier-search', {
      category: task.category,
      region: task.region
    });
    
    // Use installed skills (from marketplace)
    const quotes = await this.skills.execute('rfq-send', {
      suppliers: suppliers,
      product: task.product
    });
    
    // Compose new skill
    const negotiation = await this.skills.compose({
      name: 'negotiate-quote',
      steps: ['get-quote', 'counter-offer', 'accept-or-reject']
    });
    
    return await this.skills.execute('negotiate-quote', { quotes });
  }
}
```

## 4. The Skill Anatomy (what a skill looks like)

### Skill structure (canonical)

```typescript
interface Skill {
  // Identity
  id: string;                  // 'whatsapp-send'
  name: string;                // 'WhatsApp Send'
  version: string;             // '1.2.3'
  description: string;         // 'Send WhatsApp messages via Business API'
  
  // Categorization
  category: 'AI' | 'Commerce' | 'Business' | 'Productivity' | 'Communication' | 'Industry';
  tags: string[];              // ['messaging', 'notifications', 'customer-engagement']
  
  // Contract
  input: {
    type: 'object',
    properties: {
      to: { type: 'string' },
      message: { type: 'string' }
    },
    required: ['to', 'message']
  };
  output: {
    type: 'object',
    properties: {
      messageId: { type: 'string' },
      status: { type: 'string' }
    }
  };
  
  // Implementation
  implementation: string;      // Path to the actual code
  runtime: 'node' | 'python' | 'wasm';
  
  // Security
  permissions: string[];      // ['whatsapp:send', 'contacts:read']
  policies: string[];          // ['rate-limit-100-per-hour', 'no-spam']
  
  // Observability
  metrics: {
    executions: number;
    successRate: number;
    avgLatencyMs: number;
  };
  
  // Marketplace
  pricing?: {
    model: 'free' | 'subscription' | 'usage';
    price: number;
    currency: string;
  };
  
  // Metadata
  author: string;              // 'HOJAI Inc' or 'community'
  created: string;             // ISO date
  updated: string;             // ISO date
  source: 'built-in' | 'marketplace' | 'custom';
}
```

$ npm init -y
$ npm install @hojai/foundation

## 7. How SkillOS integrates with the 3 Pillars of HOJAI AI

```
TwinOS  ←→  MemoryOS  ←→  SkillOS
   ↑          ↑              ↑
   └──────────┴──────────────┘
              ↓
        SUTAR OS (agent runtime)
              ↓
        HOJAI AI (the products)
```

**Every skill uses all 3 pillars:**

```typescript
class ExampleSkill {
  async execute(input) {
    // 1. Load the twin (TwinOS)
    const twin = await TwinOS.get(input.entityId);
    
    // 2. Load relevant memory (MemoryOS)
    const context = await MemoryOS.read(twin.id, 'recent-context');
    
    // 3. Execute the skill (SkillOS)
    const result = await this.doWork(input, context);
    
    // 4. Update the twin (TwinOS)
    await TwinOS.update(twin.id, { lastAction: result });
    
    // 5. Store the result in memory (MemoryOS)
    await MemoryOS.write(twin.id, 'last-action', result);
    
    // 6. Learn from this execution (SkillOS learning)
    await SkillOS.learn(this.id, { input, result, success: true });
    
    return result;
  }
}
```

**The 3 pillars work together as a single capability layer.**
