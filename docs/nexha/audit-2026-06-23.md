## 8. E2E test coverage

Today there is **no automated E2E test suite** for the Nexha OS. The only end-to-end smoke test is `demos/full-stack-demo.sh` — a shell script that requires the full stack to be running, prints checkmarks, and exits non-zero on hard failures. It is **not** a regression suite.

What's needed (Phase 19):

1. **Procurement flow** — mission-planner → partner-graph → supplier-network → pricing-network → trade-finance-network → warehouse-network → distribution-network
2. **Tenant onboarding flow** — provisioning-engine → industry-tenant-instances (via webhooks) → hooks-sdk delivers events → tenant-summary reports
3. **Agent negotiation flow** — acp-messaging: QUERY → QUOTE → COUNTER → ACCEPT → ORDER → TRACK
4. **Discovery + aggregation flow** — business-directory lists agents → tenant-summary aggregates in 1 call

Each flow should:
- Run against a live stack (or a test container)
- Print ✅/❌ per step
- Exit non-zero on any failure
- Be re-runnable in CI

**Fix:** Phase 19 ships `scripts/e2e-nexha-flows.sh` + a `__tests__/e2e/` suite.
