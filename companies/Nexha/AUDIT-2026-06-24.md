## 3. Architecture Patterns

### Language split
| Language | Count | Services |
|---|---|---|
| **TypeScript** | 13 | All federation + SUTAR network services |
| **JavaScript** | 8 | All commerce/agentic + platform services |
| **TypeScript (commerce-identity)** | 1 | commerce-identity |

### Storage split
| Storage | Count | Services |
|---|---|---|
| **In-memory `Map`** | 13 | All federation + SUTAR network |
| **MongoDB + Mongoose** | 9 | acp-messaging, business-directory, mission-planner, partner-graph, commerce-runtime, provisioning-engine, hooks-sdk, tenant-summary, commerce-identity |
| **Next.js (static)** | 1 | portal |

### Auth pattern
- **JWT (HS256)** — commerce/agentic + platform services. Uses `x-internal-token` for service-to-service calls.
- **No auth** — federation + SUTAR network (these services are not directly internet-facing; called through Hub).
- **`@rtmn/shared/auth`** — required by 7 commerce/platform services.

### Shared code
- **Nexha root `shared/`** (6 packages: auth-client, auth-middleware, event-types, integration-framework, shared-types, webhook-sdk) — **all are stubs** (contain only `.env.example`).
- **`@rtmn/shared`** (at HOJAI-AI/shared) — the actual shared auth + lib used by commerce/platform services.
- **Legacy `legacy-audit/shared/`** — another copy of stubs.
