## Next.js + postcss advisory

- **Severity:** moderate (CVSS 6.1)
- **Advisory:** [GHSA-qx2v-qp2m-jg93](https://github.com/advisories/GHSA-qx2v-qp2m-jg93) — PostCSS XSS via unescaped `</style>` in CSS Stringify output
- **Affected range:** Next 9.3.4-canary.0 through 16.3.0-canary.5
- **Used in:** portal (Next 16.2.9)
- **Fix:** Upgrade Next.js (major version bump). Not feasible without breaking changes — Next 16 is bleeding edge. Documented as accepted risk.
- **Mitigation:** Portal is server-rendered Next.js. The advisory applies when PostCSS processes untrusted CSS strings — in this codebase, all CSS is author-controlled (Tailwind in globals.css + a single page.module.css). No user-supplied CSS is processed.
- **Decision:** Accept risk. Re-evaluate when Next 17 lands.
