# SUTAR OS — Pricing Model

> **Version:** 1.0
> **Date:** June 27, 2026
> **Status:** Defined

---

## Overview

SUTAR OS powers autonomous B2B commerce between AI agents. Pricing must align incentives: SUTAR earns when agents succeed, not when they fail. This document defines the complete pricing model for all SUTAR services.

---

## Core Principles

| Principle | Description |
|-----------|-------------|
| **Value-Aligned** | SUTAR earns when deal value is realized, not just when transactions occur |
| **Predictable** | Enterprises need predictable costs; agents need transparent pricing |
| **Fair** | Small businesses pay less; large enterprises pay proportionally more |
| **Usage-Based** | Costs scale with actual usage, not seat count or agent count |

## Summary

SUTAR OS pricing is designed to:

1. **Align incentives** — SUTAR earns more when deal value is higher
2. **Scale fairly** — Small businesses pay $99/month to start
3. **Be transparent** — Every component is priced explicitly
4. **Drive adoption** — Free tier and low entry barrier
5. **Be enterprise-ready** — Predictable costs at scale

The transaction fee is the primary value capture mechanism — SUTAR earns when agents succeed in creating real economic value.
